Docker安装及使用

Docker是什么？

Docker是一个快速部署、运行和管理应用的工具，可以帮助我们快速打包和部署应用。

Docker安装

去官网，https://docs.docker.com/engine/install/centos/，

使用脚本快速安装

curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh ./get-docker.sh –dry-run

docker -v查看docker版本号确定是否安装成功

systemctl start docker 启动docker

systemctl enable docker 开机自启

可视化界面portainer下载

docker volume create portainer_data 创建储存库

docker run -d -p 8000:8000 -p 9443:9443 –name portainer –restart=always -v /var/run/docker.sock:/var/run/docker.sock -v portainer_data:/data portainer/portainer-ce:latest 创建并运行portainer容器

9443端口即可访问

配置docker加速

打开阿里云 容器镜像服务ACR

sudo mkdir -p /etc/docker

sudo tee /etc/docker/daemon.json <<-‘EOF’ { “registry-mirrors”: [“https://31df3uqf.mirror.aliyuncs.com"] } EOF

sudo systemctl daemon-reload

sudo systemctl restart docker

Docker常见命令

docker run 创建并运行一个容器，-d 指后台运行，–name设置容器名称，-p设置端口映射，-e 设置环境变量

比如部署Nginx，docker run -d –name nginx -p 80:80 nginx

docker pull 镜像 拉取镜像

docker images 查看所有镜像

docker rmi 镜像名 删除镜像

docker build 构建镜像

docker save -o 包名 镜像名 打包镜像

docker load -i 包名 加载镜像

docker push 推送到镜像仓库

docker run 创建并运行容器

docker stop 容器 停止容器

docker start 容器 启动容器

docker ps 查看容器运行状态

打开.bashrc文件，设置别名

docker rm 容器 删除容器

docker logs 查看容器日志

docker exec -it 容器名 bash 进入容器

Docker数据卷

问题：容器虽然是一个操作系统，但是太简易了，甚至连vim等命令都没有，所以对一些资源的修改是非常不友好的，我们就需要用数据卷。

数据卷是一个虚拟的目录，是容器内目录与宿主机目录的桥梁。每一个数据卷都与一个真实目录关联

docker volume create 创建数据卷

docker volume ls 查看所有数据卷

docker volume inspect 查看某个数据卷详情

docker volume rm 删除某个数据卷

docker volume prune 清除数据卷

设置数据卷实现了宿主机和容器之间的双向映射比如将Nginx的html和宿主机绑定

docker run -d –name nginx -p 80:80 -v html:/usr/share/nginx/html nginx

这样在/var/lib/docker/volumes/html/_data下面就会有对应的数据，可以直接对其修改

对本地进行挂载数据卷

实现mysql数据目录、配置文件、初始化脚本的挂载

挂载/root/mysql/data到/var/lib/mysql

挂载/root/mysql/init到/docker-entrypoint-initdb.d

挂载/root/mysql/conf到/etc/mysql/conf.d

docker run -d –name mysql -p 3306:3306 -e TZ=Asia/Shanghai -e MYSQL_ROOT_PASSWORD=123

-v /root/mysql/data:/var/lib/mysql

-v /root/mysql/init:/docker-entrypoint-initdb.d

-v /root/mysql/conf:/etc/mysql/conf.d

from 指定基础镜像 from centos:7

env 设置环境变量 env key value

copy 拷贝本地文件到镜像的指定目录 copy test.tar.gz /tmp

run 执行shell命令

expose 指定容器运行时监听端口

entrypoint 启动脚本

Docker部署一个SpringBoot项目

将项目打成jar包

写Dockerfile，创建镜像，创建容器即可

Dockerfile

FROM openjdk:8
VOLUME /home/dockerTest
ADD fishexam.jar fishexam.jar
EXPOSE 8888
ENTRYPOINT [“java”,”-jar”,”/fishexam.jar”]

即基于jdk8镜像，目录为/home/dockerTest，将jar包添加到镜像中，暴露8888端口，执行命令java -jar /fishexam.jar

docker build -t fishexam .

docker run -d –name fishexam -p 8888:8888 fishexam

这样就部署了一个SpringBoot项目
