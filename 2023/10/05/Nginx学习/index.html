Nginx学习

Nginx介绍

Nginx 是一个高性能的HTTP和反向代理web服务器，常用于高并发场景，性能为王。

优点：

  • 速度快，支持高并发
  • 配置简单，扩展性强
  • 高可靠
  • 热部署

常用功能模块

  • 负载均衡
  • 反向代理
  • 动静分离

正向代理和反向代理

正向代理作用于客户端，客户端通过代理服务器来进行互联网访问，我们使用的VPN就是正向代理。

反向代理作用于服务端，用户对反向代理是无感知的，只需要将请求发送到代理服务器，代理服务器就会分发请求访问服务端，可以隐藏真实服务器。

负载均衡和动静分离

负载均衡就是通过增加服务器数量，将原本作用在一个服务器上的请求分发到多个服务器上，将负载分发，就是所谓的负载均衡。

动静分离是指将动态资源和静态资源分别部署到不同的服务器上，将静态请求和动态请求分发给不同的服务器，一般由两种方式来实现

Docker配置Nginx

创建/home/docker/nginx/conf、www、logs

先运行一个Nginx容器，将配置文件下载下来

docker run -d –name nginx -p 80:80 nginx

docker cp 容器id:/etc/nginx/nginx.conf /home/docker/nginx/conf

之后删除测试容器

docker rm -f nginx

重新创建容器，进行数据卷挂载

docker run -d -p 80:80 -p 9001:9001 –name nginx -v /home/docker/nginx/www:/usr/share/nginx/html -v /home/docker/nginx/conf/nginx.conf:/etc/nginx/nginx.conf -v /home/docker/nginx/logs:/var/log/nginx nginx

Nginx反向代理

实现效果：浏览器访问www.123.com，访问到虚拟机中的spring项目8081

  • 修改hosts中文件，配置本地dns。C:\Windows\System32\drivers\etc

  • 配置nginx.conf文件

    server {

        listen      80;
    server_name 192.168.64.128;
    location / {
        root   html;
        proxy_pass  http://192.168.64.128:8081;
        index  index.html index.htm;
    }
}

  • 重新启动Nginx就可以实现访问www.123.com就是访问8081了。

使用反向代理，根据不同的路径跳转到不同的服务端口

访问www.123.com/ssl/hello 和www.123.com/ptsnake/hello 分别对应8081和8082的springboot

server {
listen 80;
server_name 192.168.64.128;
location / {
root html;
proxy_pass http://192.168.64.128:8081;
index index.html index.htm;
}
location /ssl {
proxy_pass http://192.168.64.128:8081;
}
location /ptsnake {
proxy_pass http://192.168.64.128:8082;
}
}

Nginx负载均衡

效果：访问www.123.com/hello，这样的请求分发到8081和8082部署的两台上。

修改nginx.conf文件

upstream myserver {
server 192.168.64.128:8081;
server 192.168.64.128:8082;
}
server {
listen 80;
server_name 192.168.64.128;
location / {
root html;
proxy_pass http://myserver;
index index.html index.htm;
}
location /ssl {
proxy_pass http://192.168.64.128:8081;
}
location /ptsnake {
proxy_pass http://192.168.64.128:8082;
}
}

效果：

Nginx的负载均衡有几种方式

  • 默认使用轮询的方案，每个请求会按时间顺序逐一分配到服务器，如果服务器宕机，会自动剔除

  • weight权重策略，可以给性能较好的服务器设置较高的权重，权重设置的越高，则服务器接收的请求越多

    upstream myserver{

    ​ server 192.168.64.128:8081 weight=5;

    ​ server 192.168.64.128:8082 weight=10;

    }

  • ip_hash 这种方式会根据ip计算的hash值来访问固定的服务器，可以解决session问题

    upstream myserver{

    ​ ip_hash;

    ​ server 192.168.64.128:8081;

    ​ server 192.168.64.128:8082;

    }

  • fair 根据响应时间来分配，响应越快的优先级越高

    upstream myserver{

    ​ server 192.168.64.128:8081;

    ​ server 192.168.64.128:8082;

    ​ fair;

    }

Nginx 动静分离

将静态资源部署给Nginx实现动静分离

修改nginx.conf文件

server {
listen 80;
server_name 192.168.64.128;
location / {
root html;
proxy_pass http://myserver;
index index.html index.htm;
}
location /image/ {
root /usr/share/nginx/html/static/;
autoindex on;
}
location /html/ {
root /usr/share/nginx/html/static/;
}
location /ssl {
proxy_pass http://192.168.64.128:8081;
}
location /ptsnake {
proxy_pass http://192.168.64.128:8082;
}
}

如果路径有image或者html，就会到static目录下去寻找，这些静态资源是部署在Nginx下的，如果是用docker部署Nginx的话则是需要挂载数据卷。

解读：

conf文件中比较重要的区域就是http块，反向代理、负载均衡和动静分离都是在http中配置的。

在server块中，listen指定监听端口，server_name指定服务名称，可以是域名或者Nginx的ip，location用于配置路由，符合路由的这些请求都将转发，root指定文件夹，如果符合location的路径，则会在文件夹下寻找文件，proxy_pass用于做反向代理，将请求转发到目标服务器，index指定默认界面，实现负载均衡则是要用到upstream块，在其中配置服务ip和负载均衡策略，分别是轮询、权重、ip_hash、fair。
